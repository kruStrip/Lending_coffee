# Техническое задание для ИИ: Лендинг малого бизнеса (HTML5 + CSS3)

## Общая информация о проекте

**Цель**: Создать учебный веб-сайт (лендинг малого бизнеса) без использования JavaScript, применяя только HTML5 и CSS3.[^1]

**Тип проекта**: Лендинг малого бизнеса (например, кафе, парикмахерская, студия йоги, кондитерская, автосервис).[^1]

**Требования к структуре проекта**:[^1]

```
project/
├── index.html
├── about.html
├── contacts.html
├── css/
│   └── style.css
└── images/
```


## Технические требования

### 1. HTML-структура

**Базовая структура всех страниц**:[^1]

- `<!DOCTYPE html>` — объявление HTML5
- Корректный тег `<html lang="ru">`
- Секция `<head>` с метаданными:
    - `<meta charset="UTF-8">`
    - `<title>` — уникальный для каждой страницы
    - Подключение внешнего CSS-файла
- Секция `<body>` с контентом

**Семантическая разметка**:[^1]

- `<header>` — шапка сайта с логотипом и меню
- `<nav>` — навигационное меню
- `<main>` — основной контент (только один на странице)
- `<section>` — логические разделы контента
- `<article>` — самостоятельные блоки (отзывы, услуги)
- `<aside>` — дополнительная информация (акции, контакты)
- `<footer>` — подвал с контактами и копирайтом

**Структура контента**:[^1]

```
- Заголовки: один `<h1>` на страницу, подзаголовки `<h2>`-`<h6>` в правильной иерархии
```

- Параграфы `<p>` для текстовых блоков

```
- Выделение текста: `<strong>` для важного, `<em>` для акцентов
```

```
- Списки: `<ul>` (маркированные), `<ol>` (нумерованные), `<li>` (элементы)
```

- Ссылки: относительные между страницами проекта, абсолютные на внешние ресурсы
- Изображения: `<img>` с обязательными атрибутами `src` и `alt`[^1]

```
- Таблицы: `<table>`, `<tr>`, `<th>`, `<td>` для прайс-листа или расписания[^1]
```


### 2. Содержание страниц

**index.html** — главная страница:[^1]

- Главный заголовок с названием бизнеса
- Краткое описание услуг/продукции
- Преимущества (3-5 пунктов списком)
- Призыв к действию (кнопка/ссылка на контакты)
- Основные услуги (карточки с изображениями)
- Секция с отзывами клиентов

**about.html** — о компании:[^1]

- История создания бизнеса
- Миссия и ценности
- Команда (фото и краткие биографии)
- Достижения или сертификаты
- Маркированный и нумерованный списки

**contacts.html** — контакты:[^1]

- Адрес, телефон, email
- Форма обратной связи:
    - `<input type="text">` — имя
    - `<input type="email">` — email
    - `<textarea>` — сообщение
    - `<button type="submit">` — отправить
    - Атрибуты: `name`, `placeholder`, `required`
- Таблица с режимом работы
- Карта (можно использовать изображение или iframe Google Maps)


### 3. CSS-стилизация

**Базовые настройки**:[^1]

- Подключение внешнего файла `css/style.css`
- Сброс стилей или `box-sizing: border-box` для всех элементов
- Базовые стили для `body`: шрифт, цвет текста, фон

**Цвета и шрифты**:[^1]

- Единая цветовая схема (2-3 основных цвета)
- Контраст между текстом и фоном для читаемости
- Подключение веб-шрифта через Google Fonts
- Настройка `font-family`, `font-size`, `line-height`, `letter-spacing`
- Согласованность стилей на всех страницах

**Блочная модель**:[^1]

- Управление размерами элементов через `width`, `height`
- Настройка внутренних отступов `padding`
- Рамки `border`
- Внешние отступы `margin`
- Использование `box-sizing: border-box`

**Позиционирование**:[^1]

- `position: relative` для контейнеров
- `position: absolute` для размещения элементов внутри контейнера
- `position: fixed` для фиксированного меню сверху страницы
- `position: sticky` для заголовков секций
- Управление наложением через `z-index`

**Flexbox**:[^1]

- `display: flex` для контейнеров
- Горизонтальное меню через `flex-direction: row`
- Карточки услуг через Flexbox
- Выравнивание: `justify-content`, `align-items`
- Перенос элементов: `flex-wrap`
- Управление размерами: `flex-grow`, `flex-shrink`, `flex-basis`
- Изменение порядка: `order`

**CSS Grid**:[^1]

- `display: grid` для сложных макетов
- `grid-template-columns`, `grid-template-rows`
- Объединение ячеек: `grid-column`, `grid-row`
- Отступы между ячейками: `gap`
- Выравнивание содержимого

**Адаптивная верстка**:[^1]

- Media Queries для экранов меньше 768px (планшеты) и 480px (мобильные)
- Относительные единицы: `%`, `em`, `rem`, `vw`, `vh`
- Адаптация меню (горизонтальное → вертикальное)
- Изменение количества колонок в Grid/Flexbox
- Проверка корректности отображения на разных размерах экрана

**Анимации и переходы**:[^1]

- `transition` для плавного изменения свойств (цвет, размер, положение)
- Hover-эффекты для кнопок и ссылок
- `transform` для масштабирования, вращения элементов при наведении
- `animation` с `@keyframes` для появления элементов
- Умеренное использование эффектов (не перегружать страницу)

**Стилизация форм**:[^1]

- Отступы и размеры полей ввода
- Цвета, шрифты, рамки
- Эффекты `:hover` и `:focus` для полей
- Стилизация кнопки отправки
- Адаптивность формы


### 4. Критерии качества

**Корректность HTML-структуры**:[^1]

- Валидный HTML5-код
- Правильная иерархия заголовков
- Семантическая разметка
- Все изображения с атрибутом `alt`
- Корректные ссылки между страницами

**CSS-стилизация**:[^1]

- Применение Flexbox и Grid
- Адаптивность на всех устройствах
- Аккуратность и чистота кода
- Единообразие стилей

**Функциональность**:[^1]

- Рабочая навигация между страницами
- Корректное отображение форм
- Все интерактивные элементы работают (hover, focus)

**Визуальная эстетика и UX**:[^1]

- Гармоничная цветовая схема
- Читаемые шрифты
- Логическая структура контента
- Приятный внешний вид

## Типичные ошибки (избегать)

```
- Пропуск `<head>` или `<body>`[^1]
```

- Неверное объявление `DOCTYPE`[^1]
- Использование `<h1>` несколько раз на странице[^1]
- Неправильная последовательность заголовков[^1]
- Использование `<div>` вместо списков[^1]
- Неверные ссылки между страницами[^1]
- Отсутствие атрибута `alt` у изображений[^1]
- Некорректная структура таблицы[^1]
- Использование `<main>` несколько раз на странице[^1]
- Семантические теги без контента[^1]
- Забывают подключить CSS-файл[^1]
- Неверно указывают путь к файлу CSS[^1]
- Слишком низкий контраст текста и фона[^1]
- Использование слишком многих шрифтов[^1]
- Неверный расчет размеров с `padding` и `border`[^1]
- Отсутствие родителя с `position` для элементов `absolute`[^1]
- Не указан `display: flex` или `display: grid`[^1]
- Фиксированные размеры блоков без адаптивности[^1]
- Отсутствие атрибутов `name` у полей формы[^1]
- Кнопка без `type="submit"`[^1]
- Сломанные ссылки[^1]
