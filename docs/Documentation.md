# Техническая документация: Арома Кафе

Этот документ содержит описание внутренней архитектуры, логики работы модулей и используемых стандартов кода в проекте.

## Структура проекта

- `index.html`: Главная страница.
- `blog.html`: Страница блога с лентой статей.
- `about.html`: Информация о кофейне и команде.
- `contacts.html`: Контактные данные и форма обратной связи.
- `css/style.css`: Основной файл стилей. Использованы CSS-переменные для глобальных параметров (цвета, отступы).
- `js/catalog.js`: Логика каталога и корзины (Главная страница).
- `js/blog.js`: Логика фильтрации и комментариев (Блог).
- `js/contact.js`: Валидация форм (Контакты).

## Модуль Каталог и Корзина (`catalog.js`)

### Управление состоянием
Данные о товарах хранятся в массиве объектов `products`. Каждая позиция содержит: `id`, `name`, `price`, `category`, `image`, `sku`, `description`.

### Основные функции
- `initCatalog()`: Инициализация, рендер всех товаров.
- `renderCatalog(category)`: Фильтрация по переданной категории. Применяет CSS-классы для подсветки активного фильтра.
- `addToCart(id)`: Добавление товара в `localStorage`. Обновляет счётчик корзины в реальном времени.
- `renderCart()`: Динамический рендер модального окна корзины. Логика изменения количества с помощью кнопок `+` и `-`.
- `handleCheckout()`: Валидация формы заказа и анимация успешного завершения.

## Модуль Блог (`blog.js`)

### Чтение и Прогресс
- `readingProgress()`: Слушатель события скролла (`scroll`), вычисляющий положение на странице и регулирующий ширину верхней индикаторной полоски.
- `scrollTop()`: Функция плавного возврата в начало при клике на плавающую кнопку.

### Фильтрация контента
- Реализована на основе дата-атрибутов (`data-category`). Позволяет мгновенно переключаться между темами статей без перезагрузки.

### Комментарии
- Используется массив `comments` для хранения текущих сообщений.
- Поддерживается иерархия (возможность отвечать на конкретный комментарий) и инкрементальные лайки.

## Модуль Контакты (`contact.js`)

### Валидация форм
- Базовая проверка на заполнение обязательных полей.
- Регулярные выражения для проверки корректности E-mail.
- Анимация успеха при отправке.

## Стилизация и UI

В проекте используется единая дизайн-система, описанная в блоке `:root` файла `style.css`.
- **Primary Color (`#6F4E37`)**: Цвет кофейных зерен для акцентов.
- **Radius (`12px`)**: Универсальное скругление для всех элементов.
- **Transitions (`0.3s ease`)**: Плавность для кнопок, модальных окон и карточек.
